## 失败分析

### TC-DB5 失败根因

`trg_edgeone_deploy` 触发器已在迁移文件 `supabase/migrations/20260519000002_edgeone_auto_deploy.sql` 中新增对 `public.tools` 表的绑定，但**该迁移尚未在 Supabase 控制台执行**。

实际查询结果：`public.tools` 上仅存在 `tools_set_updated_at`（BEFORE UPDATE FOR EACH ROW）和 `trg_tools_set_updated_at`，无 `trg_edgeone_deploy`。

**修复步骤：** 在 Supabase SQL Editor 执行以下语句（幂等，可重复执行）：

```sql
DROP TRIGGER IF EXISTS trg_edgeone_deploy ON public.tools;

CREATE TRIGGER trg_edgeone_deploy
    AFTER INSERT OR UPDATE OR DELETE
    ON public.tools
    FOR EACH STATEMENT
    EXECUTE FUNCTION public.fn_notify_edgeone_deploy();
```

执行后重新运行 `npx tsx scripts/test-t3-webhook.ts` 验证 TC-DB5 和 TC-E2E4。

### TC-E2E4 结果存疑

TC-E2E4 显示 `deploy_throttle` 在 UPDATE tools 后被更新，但由于 TC-DB5 确认 `tools` 表无 `trg_edgeone_deploy` 触发器，该更新系 **TC-TH3 的 pg_net 延迟重试请求所致**（并非 tools 触发链路生效）。迁移应用后需重新验证 TC-E2E4 的真实性。
