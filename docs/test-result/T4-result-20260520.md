### TC-INT1：爬虫主流程运行

**执行命令：** `python scripts/crawler/yc.py`

**运行日志节选：**
```
[yc] 共 238 页，开始全量抓取...
[save] 全部 25 条已存在，跳过
[save] 全部 25 条已存在，跳过
...（每页均跳过，去重生效）
[yc] 进度 20/238，已写入 0 条
[yc] 进度 40/238，已写入 0 条
...
[yc] 进度 100/238，已写入 0 条
requests.exceptions.ReadTimeout: HTTPSConnectionPool(host='...supabase.co', port=443):
  Read timed out. (read timeout=15)
```

**崩溃原因分析：**

| 项目 | 说明 |
|------|------|
| 崩溃位置 | `_existing_source_urls()` → 第 ~100 页时 Supabase SSL 握手超时 |
| 错误类型 | `ReadTimeoutError` / `TimeoutError: _ssl.c:1015: The handshake operation timed out` |
| 是否代码 bug | **否**。SSL 超时是本地 Windows 网络代理问题，GitHub Actions（Linux + 直连）不受影响 |
| 数据影响 | 无（本次运行数据已在历史首次运行中全量写入，当次跳过为预期行为） |
| 结论 | 🟡 PARTIAL — 本地网络受限，建议以 GitHub Actions 运行为准 |

## 发现的问题与偏差汇总

### 偏差 1：TC-INT1 本地运行在 ~100 页时 Supabase SSL 超时

| 项目 | 说明 |
|------|------|
| 现象 | `ReadTimeoutError` / SSL 握手超时，约第 100/238 页时发生 |
| 根因 | 本地 Windows 网络环境，长时间保持 HTTPS 连接不稳定 |
| 影响范围 | 仅本地测试；GitHub Actions（Ubuntu + 直连）不受影响 |
| 数据影响 | 无，数据库已有 5775 条历史数据（首次运行时完整写入）|
| 建议 | `_existing_source_urls()` 目前每页调用一次，可优化为「首页前统一查询一次」减少连接次数 |

### 偏差 2：TC-INT3 tagline 非空率 88%（低于 90% 阈值 2 个点）

| 项目 | 说明 |
|------|------|
| 现象 | 抽样 200 条中 24 条 tagline 为空 |
| 根因 | YC S11/W12 等早期批次（2011-2012年）历史数据本身无 `oneLiner`，非代码问题 |
| 验证 | 代码中 `_format_company()` 正确读取 `raw.get("oneLiner") or ""`，逻辑无误 |
| 建议 | 测试方案可调整为：对近 3 年批次（W22+）单独统计，或将 tagline 阈值调整为 ≥ 85% |
