## 一、相对 v1.1 的关键修订

### 1.1 砍掉了什么（无登录引发）

| v1.1 模块 | 命运 | 原因 |
|---|---|---|
| 阿里云短信 + 验证码 + JWT 中间件 | ❌ 删 | 无用户体系 |
| 收藏 / 取消收藏 / 我的收藏 | ❌ 删（或退化为 localStorage） | 无账号 |
| 个人中心 / 我的项目 | ❌ 删 | 同上 |
| 环信 IM 1v1 咨询（W3 整周） | ❌ 删 | 匿名做不了点对点 IM |
| 数据库表 `users` `messages` | ❌ 删 | 没用户也没消息 |

### 1.2 改造了什么

| v1.1 模块 | 改造后 |
|---|---|
| 用户登录后发布项目 → 审核 | **匿名表单** → Edge Function 写 `submissions` 表 → 飞书通知主理人 |
| Vue + Antd Pro 后台 | **Next `/admin?token=xxx`**（URL token 鉴权，无登录） |

### 1.3 技术栈完全替换

| v1.1 选型 | v2.0 选型 | 替换原因 |
|---|---|---|
| FastAPI + 自建 PG | Supabase（Postgres + RLS + Edge Functions） | 跟邓晖团队 xiaomo-rd-ai 主栈对齐 |
| Vue 3 + Antd Pro | **沿用现有 Next 16 + React 19 + Tailwind v4** | 现有 23 路由不能扔 |
| Celery + Redis | GitHub Actions cron | 0 运维 |
| JWT + 阿里云短信 | 不需要 | 无登录 |
| Docker Compose + 单机 + Nginx | **EdgeOne Pages**（静态托管 + 边缘函数） | 0 运维 |
| Uptime Kuma | EdgeOne 自带监控 + 飞书告警 | 同上 |

## 二、目标架构（EdgeOne + Supabase + GitHub Actions 三件套）

```
                ┌────────────────────────┐
   主理人 ────►│  Next /admin?token=xxx │── PATCH ──┐
                └────────────────────────┘            │
                          ▲                            ▼
                          │ 通过/驳回                ┌──────────────────┐
                          │                          │   Supabase       │
   匿名访客 ──表单──┐    │                          │  ┌────────────┐  │
                    │    │                          │  │ tools      │  │
                    ▼    │                          │  │ submissions│  │
            ┌────────────────┐    INSERT            │  │ subscriptions│ │
            │ EdgeOne Edge Fn│────────────────────► │  └────────────┘  │
            │  (V8 isolate)  │                      └──────────────────┘
            └────────────────┘                              │
                                                             │ webhook
                                                             ▼
   GH Actions cron ──Python 爬虫──► Supabase REST           ┌──────────────┐
   (每日 2:00 跑 YC/PH)             (写 submissions pending) │ EdgeOne Pages│
                                                             │ 重新构建      │
                                                             │ (Next SSG)   │
                                                             └──────┬───────┘
                                                                    │
                                                                    ▼
                                                              moxie.ai
                                                              (静态托管)
```

**三个角色的运行时位置**：
- **静态站**：Next.js 16 `output: 'export'`，build 时从 Supabase 拉所有 `tools where status=approved` → 出静态 HTML → 推 EdgeOne Pages
- **写操作**：访客提交 / 主理人审核 / Newsletter 订阅 → EdgeOne Edge Function（TS）→ Supabase REST（用 service key）
- **爬虫**：GitHub Actions cron（不在边缘也不在 Supabase）→ 直接 POST Supabase REST 写 submissions

## 三、数据模型

### 3.1 三张表

```sql
-- 主理人策展工具（现有 src/lib/data.ts 24 条迁这里）
create table tools (
  slug text primary key,
  name text not null,
  name_en text,
  tagline text not null,
  description text,
  level text check (level in ('L1','L2','L3','L4')),
  rating int,
  category text not null,
  tags text[],
  pricing text check (pricing in ('free','freemium','paid')),
  price_note text,
  zimo_view text,                  -- 主理人主观评价
  good_for text[],
  not_good_for text[],
  website_url text not null,
  affiliate_url text,
  video_url text,
  video_title text,
  logo_url text,
  cover_url text,
  published_at date,
  updated_at timestamptz default now(),
  is_sponsored boolean default false,
  status text default 'approved' check (status in ('approved','draft','archived')),
  source text default 'curation'   -- curation / submission / crawler
);

-- UGC 提交 + 爬虫数据（待主理人审）
create table submissions (
  id uuid primary key default gen_random_uuid(),
  payload jsonb not null,          -- 提交的工具字段（slug 可空，approve 时主理人补）
  source text not null,            -- 'anonymous_form' / 'crawler:yc' / 'crawler:ph'
  submitter_email text,            -- 匿名表单可填可不填
  status text default 'pending' check (status in ('pending','approved','rejected')),
  reject_reason text,
  created_at timestamptz default now(),
  reviewed_at timestamptz,
  approved_tool_slug text references tools(slug)
);

-- Newsletter 订阅
create table subscriptions (
  email text primary key,
  source text,                     -- 'sidebar' / 'footer' / 'submit_page'
  created_at timestamptz default now(),
  confirmed boolean default false
);
```

### 3.2 RLS 策略

- `tools`：anon 只能 SELECT `status=approved`；service key 全权限
- `submissions`：anon 不能读；anon 可以 INSERT；service key 全权限
- `subscriptions`：anon 可以 INSERT 自己的 email；anon 不能读

写操作不经 RLS 走 Edge Function + service key，避免暴露 anon key 时被滥用。

### 3.3 跟现有 `Tool` TS 类型的映射

字段几乎 1:1，需要做的：
- snake_case ↔ camelCase 在 `src/lib/data.ts` 的拉取层做转换
- `Tool` 类型继续用，data 来源从 const 数组改成 build 时 Supabase fetch
- `status` `source` 是新加的，前端 SELECT 时只取 `status=approved` 就行

### 4.1.1 爬虫架构演进路径（重要：避免过度设计 + 保留升级空间）

**MVP 阶段（1 个源）**：函数级分层，**不**上 abstract class。`scripts/crawler/yc.py` 内部分 4 段：

```python
def fetch(url, retry=3) -> str           # 共用：HTTP get + UA 轮换 + 限速 + 重试
def parse_yc(html) -> list[dict]          # YC 专属：BeautifulSoup 解析
def normalize(raw) -> dict                # 共用：→ {name, tagline, desc, website_url, source_url}
def push_submissions(items, source) -> int  # 共用：写 Supabase submissions 表 + 失败飞书告警
```

`fetch / normalize / push` 单独放 `scripts/crawler/_common.py`，`parse_yc` 留在 `yc.py`。

**v2.1+ 第 2 个源（PH/HF）进来时升级**：抽 `BaseCrawler` abstract class，`YCCrawler/PHCrawler` 实现 `parse()`，其他方法继承。预估升级成本 **1.5h**，比"MVP 就上 abstract class"的 2h 还省，且基于真实差异决策不靠猜。

## 五、EdgeOne 关键约束（避免踩坑）

| 想做的事 | 能不能在 EdgeOne 上做 | 替代 |
|---|---|---|
| Next.js `output: 'export'`（纯 SSG） | ✅ | — |
| Server Actions / Node API Routes | ❌ 边缘函数是 V8 isolate，不是 Node | EdgeOne Edge Function（TS/JS） |
| Next.js ISR / on-demand revalidate | ⚠️ 受限 | 改 Supabase webhook → EdgeOne 重新构建 |
| 跑 Python / Crawl4AI / Playwright | ❌ | 爬虫必须放外面 → GitHub Actions |
| 在 Edge Fn 里 fetch Supabase REST | ✅ | 用 service key，别在前端暴露 |
| 边缘函数 CPU / 内存 | < 50ms CPU / 128MB | 重活儿走 Supabase Edge Functions（Deno）或 GH Actions |
